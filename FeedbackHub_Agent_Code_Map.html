🎓 FeedbackHub v4 - Agent Code Map & Architecture

📋 Project Overview

FeedbackHub v4 is a comprehensive trainee feedback and performance tracking system with AI-powered insights.

Tech Stack:

  • Backend: Java Spring Boot, Spring Security, JPA/Hibernate, MySQL
  • Frontend: Angular 15+, TypeScript, Angular Material, RxJS
  • AI Integration: OpenAI API (for feedback & skills gap analysis)

🔐 Authentication & Security

Login Flow

User Login → AuthController.login()
    ↓
UserDetailsService.loadUserByUsername()
    ↓
Generate JWT Token
    ↓
Return { token, userId, role, fullName, email }
    ↓
Frontend stores token in localStorage
    ↓
All subsequent requests include: Authorization: Bearer {token}
POST /api/auth/login
Body: { username, password }
Returns: AuthResponse with JWT token
Frontend: LoginComponent → AuthService → JwtInterceptor

👨‍🏫 Trainer Features

1. Trainer Dashboard

Overview of all trainees, class performance, at-risk students, and pod analytics.

GET /api/trainer/dashboard
Returns: DashboardDto with:

View pods, students in each pod, and individual student progress with AI insights.

Upload Excel files to import scores or create cohorts/pods.

POST /api/scores/bulk-upload
Body: MultipartFile (Excel)
Returns: BulkUploadResult with success/error counts
POST /api/cohort/upload NEW
Body: MultipartFile (Excel with student details)
Returns: Cohort creation result
GET /api/template/score-upload NEW
Returns: Excel template file for download
Frontend: BulkUploadComponent, DownloadTemplateComponent, CohortUploadComponent

4. Student Directory

View all trainees with search, filter, and sorting capabilities.

Component: StudentDirectoryComponent
Features: Search by name/email, filter by pod/cohort, sort by score/name

5. Activity Log

Track all system activities (uploads, feedback, score submissions).

GET /api/activity/recent?limit=50
Returns: List of recent activity logs
Component: ActivityLogComponent
Backend: ActivityLogService.getRecentActivity()

🤖 AI Agent Integration

AI Integration Points

1. Score-Level AI Feedback

Use Case: When a trainee views their score, AI provides personalized feedback on strengths and areas to improve.

Backend: AiFeedbackController.getAiFeedback(scoreId)
→ Fetch score details from DB
→ Call OpenAI API with prompt: "Analyze this score and provide constructive feedback..."
→ Return AI-generated text
Frontend: <app-ai-feedback [feedback]="aiFeedback"></app-ai-feedback>
Displays AI feedback with a robot icon

2. Skills Gap Analysis

Use Case: Trainee or trainer views a student's overall profile; AI identifies weak categories and suggests action plans.

Backend: AiFeedbackController.getSkillsGap(traineeId)
→ Fetch all scores for trainee
→ Group by category, calculate averages
→ Call OpenAI API: "Identify weak areas based on these scores..."
→ Return structured analysis
Frontend: <app-skills-gap [analysis]="skillsGapAnalysis"></app-skills-gap>
Displays weak categories with improvement tips

3. Consolidated Report (Trainer Agent)

Use Case: Trainer generates a class-wide AI report with insights, trends, and recommendations.

Endpoint: GET /api/trainer/agent/consolidated-report
Returns:
  • Class summary
  • Per-student summaries with status (STRONG, AT_RISK, etc.)
  • Category insights (which categories need attention)
  • Recommendations for trainers
Component: ConsolidatedReportComponent

AI Implementation Guide

1. Backend Integration (Example): 
// AiFeedbackController.java
@GetMapping("/feedback/{scoreId}")
public ResponseEntity<String> getAiFeedback(@PathVariable Long scoreId) {
    Score score = scoreRepository.findById(scoreId).orElseThrow(...);
    String prompt = buildPrompt(score); // Include score, category, assignment
    String aiFeedback = openAiService.generateFeedback(prompt);
    return ResponseEntity.ok(aiFeedback);
}

// OpenAiService.java
public String generateFeedback(String prompt) {
    // Call OpenAI API
    ChatCompletionRequest request = ChatCompletionRequest.builder()
        .model("gpt-4")
        .messages(List.of(new Message("user", prompt)))
        .build();
    return openAiClient.createChatCompletion(request).getChoices().get(0).getMessage().getContent();
}
2. Frontend Integration (Example): 
// trainee-dashboard.component.ts
ngOnInit(): void {
    this.dashSvc.getTraineeDashboard(userId).subscribe(stats => {
        this.stats = stats;
    });
    
    // Fetch AI feedback separately
    this.aiService.getAiFeedback(scoreId).subscribe(feedback => {
        this.aiFeedback = feedback;
    });
    
    this.aiService.getSkillsGap(userId).subscribe(analysis => {
        this.skillsGap = analysis;
    });
}

📊 Data Flow Examples

Example 1: Trainer Views Dashboard

1. User logs in as Trainer
2. Frontend navigates to /trainer/dashboard
3. TrainerDashboardComponent.ngOnInit() calls:
   → DashboardService.getTrainerDashboard()
4. HTTP GET /api/trainer/dashboard (with JWT token)
5. Backend: JwtAuthenticationFilter validates token
6. DashboardController.trainerDashboard() invoked
7. DashboardService.getTrainerDashboard(email) called
8. Service queries:
   → UserRepository.findByRole(TRAINEE) - get all trainees
   → ScoreRepository.findAvgScoreByTrainee() - for each trainee
   → ScoreRepository.findAvgScoreByPod() - for each pod
   → ActivityLogService.getRecentActivity(10)
9. Build DashboardDto with all aggregated data
10. Return to frontend
11. TrainerDashboardComponent displays:
    → KPI cards (total students, avg score, at-risk count, active pods)
    → Pod performance bar chart
    → At-risk students list with AI feedback
    → Recent activity log

Example 2: Trainee Views AI Feedback on Score

1. Trainee navigates to Scorecards page
2. ScorecardsComponent loads all scores
3. For each score, component requests AI feedback:
   → AiService.getAiFeedback(scoreId)
4. HTTP GET /api/ai/feedback/{scoreId}
5. Backend: AiFeedbackController.getAiFeedback(scoreId)
6. Service:
   → Fetch Score entity from DB
   → Build prompt: "Provide feedback for this score: ..."
   → Call OpenAI API
   → Return AI-generated feedback text
7. Frontend receives feedback
8. AiFeedbackComponent displays feedback with robot icon

Example 3: Bulk Score Upload

1. Trainer uploads Excel file via BulkUploadComponent
2. User can download template first using DownloadTemplateComponent
3. File selected → ScoreService.bulkUpload(file)
4. HTTP POST /api/scores/bulk-upload (multipart/form-data)
5. Backend: ScoreController.bulkUpload(file)
6. Service:
   → Parse Excel using Apache POI
   → Validate each row (email, score range, etc.)
   → For each valid row:
      • Find User by email
      • Create Score entity
      • Save to DB
      • Log activity
   → Collect errors for invalid rows
7. Return BulkUploadResult { successCount, errorCount, errors[], uploaded[] }
8. Frontend displays results table with success/error counts

🔧 Configuration & Setup

Backend Configuration

application.properties: 
# Database
spring.datasource.url=jdbc:mysql://localhost:3306/feedbackhub
spring.datasource.username=root
spring.datasource.password=yourpassword

# JPA
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true

# Security
jwt.secret=your-secret-key
jwt.expiration=86400000

# AI (if using OpenAI)
openai.api.key=sk-...
openai.model=gpt-4

Frontend Configuration

environment.ts: 
export const environment = {
    production: false,
    apiUrl: 'http://localhost:8080/api'
};

Backend:

cd frontend
